# chn-wc26na-hub

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能包含信息展示、工具界面、说明文档等不同用途，每个页面在逻辑上彼此独立，不依赖特定域名或外部服务。

## 项目简介

这是一个通用型 HTML 页面归档仓库。所有页面均以静态 HTML 文件形式存放，可直接通过浏览器打开查看。仓库本身不绑定任何特定网站或应用，仅作为页面内容的集中存储与版本管理。

## 目录说明

```
.
├── index.html          # 仓库首页 / 页面索引（可选）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 页面共用的资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **`pages/`**：每个 HTML 文件代表一个独立的页面，文件名应尽量反映页面内容或用途。
- **`assets/`**：存放多个页面可能共享的样式、脚本或图片资源，避免重复。

## 页面归档说明

- 所有页面均为纯静态 HTML，无需后端支持。
- 页面之间无硬性依赖关系，可单独访问。
- 若页面引用了外部资源（如字体、库文件），建议优先使用本地或 CDN 通用链接，并注明来源。
- 页面内容应保持简洁、可读，避免包含任何商业推广或诱导性信息。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并确保文件名不与已有文件冲突。
- 若需修改共享资源，请在 `assets/` 对应子目录中更新，并注意兼容性。
- 不建议在页面中嵌入直接跳转至特定域名的链接或脚本。
- 欢迎通过 Issue 或 Pull Request 提交改进建议，但请确保内容符合仓库通用归档性质。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接双击 `pages/` 下的任一 HTML 文件，即可在浏览器中查看。
3. 也可将整个仓库部署到任意静态托管服务（如 GitHub Pages），通过 URL 访问页面。
